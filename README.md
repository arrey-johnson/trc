# The Reset Circle App

A mobile-first PWA that replaces manual WhatsApp accountability rituals with structured check-ins — while keeping WhatsApp as the trusted "proof of accountability" layer.

## Phase 1 (current)

- Email + password auth (Supabase) — signup with name, email, WhatsApp number
- Onboarding: editable morning/evening routines, reminder times
- Daily check-in with explicit ✅/❌ per item
- Formatted WhatsApp report + one-tap share (`navigator.share` or `wa.me` fallback)
- PWA manifest (installable)
- **Push notifications** — morning/evening reminders, follow-ups, forum posts
- In-app notification inbox (`/notifications`)

**Built:** history/streaks UI (`/progress`), admin dashboard (`/admin`, Phase 4a).

**Not yet built:** broadcast notifications, admin role UI, export.

## Stack

- Next.js 14 (App Router) + TypeScript + Tailwind
- Supabase (Postgres, Auth, RLS)
- Vercel (hosting + cron for push reminders)

Default timezone: `Africa/Douala` (WAT, UTC+1).

## Setup

### 1. Supabase project

1. Create a project at [supabase.com](https://supabase.com).
2. **Authentication → Providers → Email** — ensure email auth is enabled.
3. For faster local testing, disable **Confirm email** under Email provider settings (optional).
4. **SQL Editor** — run migrations in order:
   - `supabase/migrations/001_initial_schema.sql`
   - `supabase/migrations/004_forum_posts.sql`
   - `supabase/migrations/005_notifications.sql`
   - `supabase/migrations/006_admin_rls.sql`
   Or run the combined `supabase/setup.sql` on a fresh project.
5. Copy your project URL and anon key from **Settings → API**.

### 2. Environment

```bash
cp .env.example .env.local
# Fill in Supabase URL, anon key, service role key, VAPID keys, CRON_SECRET
npm run generate-vapid   # prints VAPID + CRON_SECRET to paste into .env.local
```

### Push notifications (local + production)

1. Run migration `005_notifications.sql` in Supabase SQL Editor.
2. Generate keys: `npm run generate-vapid` — paste output into `.env.local` and Vercel env vars.
3. Add **service role key** from Supabase → Settings → API (`SUPABASE_SERVICE_ROLE_KEY`, server-only).
4. Verify: `npm run verify-push` (loads from `.env.local` if you use a dotenv loader, or export vars first).
5. On a phone: **Add to Home Screen** → open from home screen → Settings → **Enable notifications**.

**Vercel production (required for server-side push):**

| Variable | Notes |
|----------|--------|
| `NEXT_PUBLIC_VAPID_PUBLIC_KEY` | From `npm run generate-vapid` |
| `VAPID_PRIVATE_KEY` | Server-only, never expose to client |
| `VAPID_SUBJECT` | `mailto:you@yourdomain.com` |
| `CRON_SECRET` | Same value Vercel sends as `Authorization: Bearer …` |
| `SUPABASE_SERVICE_ROLE_KEY` | Sends push + writes notification rows |

Cron hits `/api/cron/reminders` on the schedule in `vercel.json` (hourly on Pro; **once per day on Hobby**). Each run sends reminders whose local time has passed, including catch-up for yesterday if the previous run was too early.

Test production cron after deploy:

```bash
curl -H "Authorization: Bearer YOUR_CRON_SECRET" https://your-app.vercel.app/api/cron/reminders
```

Expect `{ "ok": true, "config": { "pushReady": true, ... } }` — not a redirect to login.

**iOS:** Web Push requires iOS 16.4+ and the app **installed to the home screen** (Safari tab alone will not work).

**Hobby plan tip:** Vercel only triggers cron once daily. For hourly reminders globally, upgrade to Pro or ping the same endpoint hourly from an external cron service using your `CRON_SECRET`.

### 3. Run locally

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

### 4. PWA icons

Replace placeholder icons in `public/icons/` with your brand assets (192×192 and 512×512 PNG).

## User flow

1. **Sign up** — name, email, WhatsApp number, password → `/auth/signup`
2. **Sign in** — email + password → `/auth/login`
3. **Onboard** — morning/evening routine items (editable templates), reminder times
4. **Home** — see today's morning/evening status
5. **Check in** — tap ✅ or ❌ for every item; ❌ requires a short reason
6. **Share** — copy or send formatted report to your WhatsApp group

## Report format

Generated by `src/lib/report.ts` — edit that file to tweak wording.

```sql
update public.users

```

## Deploy (Vercel)

```bash
vercel
```

Add the same env vars in the Vercel project settings (including `SUPABASE_SERVICE_ROLE_KEY`, VAPID keys, and `CRON_SECRET`). Vercel Cron is configured in `vercel.json`.

## Notification types

| Type | When |
|------|------|
| Morning / evening reminder | At your set times if today's check-in isn't logged |
| Follow-up | ~2 hours after first reminder, still no check-in |
| Forum post | When an admin publishes a new post |

All notifications appear in `/notifications` and as push alerts on subscribed devices.
